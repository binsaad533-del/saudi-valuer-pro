
## 🎯 تحويل رقيم من مساعد محادثة → وكيل تنفيذي

### المرحلة 1: البنية التحتية للأوامر التنفيذية
**ماذا**: نظام "Action Tokens" موحد في Edge Function `raqeem-chat` و `raqeem-client-chat` يكتشف نوايا المستخدم ويترجمها لأوامر تنفيذية.

**الأوامر المدعومة لكل دور:**

#### المالك/المقيّم:
- `[ACTION:CHANGE_STATUS]` — تغيير حالة الطلب (عبر `update_request_status`)
- `[ACTION:ASSIGN_INSPECTOR]` — تعيين معاين لطلب
- `[ACTION:GENERATE_INVOICE]` — إصدار فاتورة
- `[ACTION:APPROVE_DRAFT]` — اعتماد المسودة
- `[ACTION:REJECT_DRAFT]` — إعادة المسودة مع ملاحظات
- `[ACTION:WEEKLY_REPORT]` — تقرير أداء فوري

#### العميل:
- `[ACTION:TRACK_REQUEST]` — تتبع حالة الطلب
- `[ACTION:UPLOAD_DOCUMENT]` — رفع مستند إضافي
- `[ACTION:REQUEST_EDIT]` — طلب تعديل رسمي
- `[ACTION:PAY_INVOICE]` — رابط دفع مباشر

#### المعاين:
- `[ACTION:MY_TASKS]` — عرض المهام المسندة
- `[ACTION:SUBMIT_INSPECTION]` — تسليم تقرير المعاينة
- `[ACTION:UPLOAD_PHOTOS]` — رفع صور المعاينة

#### المدير المالي:
- `[ACTION:CONFIRM_PAYMENT]` — تأكيد استلام دفعة
- `[ACTION:OVERDUE_REPORT]` — تقرير المتأخرات
- `[ACTION:REVENUE_SUMMARY]` — ملخص الإيرادات

### المرحلة 2: تحسين واجهة الدردشة
- أزرار إجراءات سريعة ديناميكية حسب الدور
- عرض نتائج الأوامر كبطاقات تفاعلية (ليس نص فقط)
- تأكيد قبل تنفيذ الأوامر الحساسة (تغيير الحالة، الدفع)

### المرحلة 3: الذكاء الاستباقي
- رقيم يقترح إجراءات بناءً على السياق
- تنبيهات فورية داخل الدردشة
- ملخصات دورية تلقائية

### ⚠️ الحوكمة:
- كل أمر تنفيذي يمر عبر `update_request_status` (لا تجاوز)
- تسجيل كل إجراء في `audit_logs`
- تأكيد المستخدم قبل تنفيذ أي أمر حساس
